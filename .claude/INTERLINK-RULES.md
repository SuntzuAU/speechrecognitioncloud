# Interlink Rules — SRC Is NOT a Gateway Site

## SRC Linking Policy

SpeechRecognition.cloud is the SaaS product. It is NOT part of the gateway cross-linking network.

### Outbound links from SRC
- Link to **voicerecognition.com.au ONLY** as the parent brand/authority site
- Do NOT link to other gateway sites (dp16, DNS, dictation, dmo, pdfsoftware, cloudprinting)
- SRC stands on its own as an independent product
- Anchor text to VRA: use descriptive terms like "Voice Recognition Australia" or "25 years of speech recognition expertise"

### Inbound links TO SRC (from gateway sites)
- Gateway sites may link to SRC where contextually relevant
- Approved anchor text: "best free speech to text", "cloud speech recognition", "Speech Recognition Cloud"
- SRC does not need to reciprocate these links

### Why this policy exists
- SRC targets international users (USD pricing). Gateway sites target Australian buyers.
- Cross-linking SRC into the Australian gateway network dilutes its international positioning
- SRC should build its own authority independently, not borrow from the gateway network
- The only connection back to VRA is the parent brand credibility link

## Internal Links (within SRC)

- Blog posts should include 1-2 internal links to other SRC pages (use cases, pricing, support)
- No footer links — all links contextual in body copy
- Use descriptive anchor text, not "click here"

## Blog Post Frontmatter

```yaml
internalLinks:
  - to: "/pricing"
    anchor: "speech recognition pricing plans"
externalLinks:
  - to: "voicerecognition.com.au"
    anchor: "Voice Recognition Australia"
    url: "https://www.voicerecognition.com.au"
```

Suggest 2-3 options, let owner choose. Never commit without approval.
